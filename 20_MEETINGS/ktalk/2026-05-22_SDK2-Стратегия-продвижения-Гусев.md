## Краткое содержание

Обсуждение стратегии продвижения SDK2 клиентам. Формирование позиционирования, списка ценностей, плана миграции.

## Основные темы

### Проблемы SDK1
- Устаревший внешний вид (UI)
- Долго и дорого разрабатывать, чинить баги
- Нет автотестов — низкая стабильность
- Отсутствие дизайн-системы

### Ценности SDK2 для клиента
- Обновлённый UI, swipe-жесты (как в Telegram)
- Дизайн-система: 2-3 цвета клиента — SDK перекрашивается
- Большое покрытие автотестами — выше стабильность
- Кастомизация кнопок через API
- Документация в Markdown — можно использовать AI-агентов для встраивания
- Возможность писать автотесты на встроенный SDK

### План продвижения клиентам
- Сбор фич, запрошенных клиентами (Магнит, ЦБ), но не реализованных
- Сравнение SDK1 vs SDK2
- Формирование роадмапа развития
- Мотивация: прекращение поддержки SDK1 через год + новые фичи только в SDK2

### Демо-приложение
- Выпуск в App Store и Google Play
- Возможность подключения к своему чат-серверу
- Потенциально: настройка цветовой темы в демо (3-5 дней)

### Политика лицензирования
- Новая лицензия на SDK2 (~10 млн)
- Лицензия апгрейда (~в 2 раза дешевле для текущих клиентов)

### Процесс миграции
- Возможно нужен свежий чат-сервер (уточнить)
- A/B тестирование — рекомендуемый путь
- Встроить обе версии SDK в приложение, переключать через сервер
