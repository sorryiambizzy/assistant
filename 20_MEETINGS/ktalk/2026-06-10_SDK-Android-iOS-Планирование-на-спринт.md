## Краткое содержание

Спринт-планирование по мобильным SDK. Разобрали хвосты SDK 1 (iOS-баг ЦБ, поднятие мажора), статус тестирования SDK 2 (Android 0.4.0 и iOS 0.5.0), и ключевой блокер по VoIP-звонилке — отсутствие API-контракта со стороны НЦЦ/МФЦ. Отдельно — статус киллер-фич (ждём Гусева) и приветственного сообщения.

## Основные темы

### SDK 1
- **iOS, баг ЦБ (клиентский):** подрядчик, который внедряет, сам пошёл править → не к спеху. Сергей заведёт отдельный **хотфикс**, выпуск по стандартному процессу.
- **Поднятие мажорной версии:** задача готова, нужно перевести в MD. Срок «до 1 июня» фигурировал в письме, но по сути это история для нас, не для клиентов (рисков от задержки нет). **Решение:** сделать в июне, раз запланировали и обозначили срок. Тестировщиков подключаем, когда освободятся (Настя по процессу подхватит после Android).
- **Android SDK 1:** активных задач нет.

### VoIP / звонилка (задача 1086) — ключевой блокер
- Иван в рамках исследования ответил: технология **WebRTC применима** на мобильных SDK и **проще в реализации, чем SIP**; проверил на текущем функционале НЦЦ.
- **Проблема:** нет API-контракта. Матушкин нарисовал схему/архитектуру со стороны МФЦ, но Иван не может её интерпретировать; Матушкин **не берёт ответственность** двигать активность дальше.
- **Что можно начать без бэка:** UI-части кастомизации звонилки (экран активного звонка, mute и т.п.), частично — функционал Пуша (данные для подключения в пуше). Но без бэка функционал не завершить; есть риск сделать работу впустую, если бэк не поддержит задуманное на этой технологии.
- **Нужно:** спецификация API от НЦЦ/МФЦ (методы, действия, протокол взаимодействия).
- **Решение Марка:** перевести задачу в статус **«требуется информация для выяснения и доработки аналитики»** — задачи не готовы. Активность падает на аналитиков (Аня Меняева проработала постановку, но это постановка, не реализация со стороны МФЦ). Марк завтра разбирается.
- **Вычитка постановки:** вместо Ивана — **Олег Анофонтов** (в контексте); Иван пропишет в Mattermost.
