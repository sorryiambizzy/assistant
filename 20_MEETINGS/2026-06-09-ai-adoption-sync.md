## Ключевые решения и апдейты

### 1. Documentation / Gramax — заливка доки и доступ MCP

- **Ирина залила доку SMP в Gramax** (с нюансами вёрстки, на содержание не влияет). **Без** разделов: настройки Дашбордов, диаграмма Ганта, мобильное приложение — эти 3 модуля + мобильное вынесены в отдельную документацию. Тестим пока без них; доку мобильного приложения Ирина закинет при необходимости.
- **MCP до Gramax ходит в портал Knowledge, а не в Editor Nau** (где сейчас лежит дока, нет токена). Решение: **залить доку в Knowledge Nau** — это не критично. Push при сложностях — через **Макса** или **Дениса Кирпиченкова**.
- **У Антона нет MCP Gramax.** Ирина пришлёт инструкцию по настройке; в MCP можно указать **конкретный каталог** для поиска (сузить до загруженного раздела). После заливки — отпишется в чат + скинет инструкцию.
- **План Ирины:** залить доку на Knowledge → настроить коннект доки с плагином Антона.
- ⚠️ Антон: ранее с MCP результат был **хуже, чем локально** — нужно проверить, как MCP до Gramax отрабатывает поиск.
- **Четверг (11.06):** командные практики отдела аналитики (под Юлей Шаровой) — Ирина расскажет, как работает со своим ассистентом.

### 2. BPM-помощник (Антон) — масштабирование

- Доработал тест-кейсы (добавил вводных в сложные), перетестировал. **Сложные кейсы прогнал на GLM-4.5** (раньше всё на GLM-4.6/4.7).
- С доп. контекстом + сильной моделью **почти все тест-кейсы пройдены**; самый сложный — частично (концептуально ок, но есть неточности, способные ввести в заблуждение). Контекста добавлял минимум (1 строка / пара предложений).
- **Рецепт масштабирования:** локальная выгрузка доки по модулям → для каждого модуля плагин + скил по **готовому шаблону**. По мнению Антона, **решением уже можно пользоваться**, нужно расширить на другие ВП и на доку SMRM (которую готовит Ира).
- Под Gramax скил надо **переадаптировать** (сейчас заточен на локальный поиск) — недолго.
- **Токены:** на GLM-4.5 к среде токены кончаются. Рекомендация: простые вопросы — 4.6/4.7, сложные — 4.5. **Решение ЕП: катаем 4.5.**
- Антон **запушил** обновлённый скил (минорные правки); **готовит отчёт по всем тест-кейсам** — вышлет ЕП.

### 3. Ask Experts — ВП-скил Даши (поиск БТ по АСКу)

- Даша сделала ВП-скил: **БТ находит нестабильно**. Расхождение по кейсу: искать **связанные** БТ или **корневые/головные** (типа «доработка мобильного приложения», «улучшение интерфейса»).
- **Решение:** головные/корневые БТ **отбрасываем** — это фича для вложения, не нужна (Ирина). Целевой кейс — поиск **реальных кейсовых БТ** по связке «название + описание + фичи АСКа». Что делать с найденным — пока не важно.
- **Тестирование:** на завершённых тасках БТ уже заведены → Даше **скормить синтетический/фейковый АСК** (название + описание + фичи). Обкатывать на **базовых** кейсах, не на краевых.
- Тестит только на **мобильных и SMP** АСКах (по ВП не в контексте).
- **Интеграция:** скил позже **вольётся апдейтом в ассистента аналитика** Ирины. Сначала докатать и залить как скил.
- Даше: подготовить **краткие итоги** (что получается / что нет) для ЕП.
