### 4. Демо-витрина SDK 2 (для Гусева / магнита)

- **Отдельное приложение** (не путать с SDK demo)
- **Подход с ключами:**
  - Приватный ключ зашит в сборку приложения
  - Публичный ключ передаётся клиенту в инструкции вместе со ссылкой на TestFlight
  - Клиент вставляет публичный ключ в **настройках витрины** своего стенда
  - Запрашивать у клиента генерацию ключей — некорректно
- **Diplinks + QR-коды** (Иван подсмотрел реализацию в Codex): QR со ссылкой на тестовый стенд, приложение раскладывает параметры (адрес сервера, CRM ID, витрина) в настройки
- **Поток для нового клиента:** ссылка на TestFlight + предзашитые данные демо-витрины → "открыть чат" → попал в чат
- **Поток для текущего клиента:** ссылка + публичный ключ + адрес своего стенда → клиент настраивает витрину у себя → подключается
- **Catя Коновалова** — подключаем как аналитика на UX проработку
- **Кастомизация витрины** (цвета и т.д.) уже есть в Chat UI Kit, времени немного. Делаем итерационно: сначала базовая аутентификация → потом кастомизация
