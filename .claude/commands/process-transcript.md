# /process-transcript - Обработка встреч

> **Интерактивная обработка транскриптов из Kontur Talk или файлов**

---

## Что делает эта команда

1. Получает транскрипт (MCP ktalk или файл из чата)
2. Анализирует и извлекает сущности
3. Уточняет неясные моменты
4. Распределяет по vault по правилам
5. Проводит финальный диалог

---

## Использование

### Вариант A: Через MCP ktalk
```
Пользователь: Добавь последнюю встречу в базу знаний
```

### Вариант B: Через файл
```
Пользователь: [загружает файл transcript.txt]
Пользователь: Добавь эту встречу в базу знаний
```

---

## Алгоритм работы

### Шаг 1: Получение транскрипта

**Если MCP ktalk:**
1. Получить список записей через ktalk MCP
2. Показать последние 5-10 записей
3. Пользователь выбирает нужную

**Если файл:**
1. Прочитать содержимое файла из чата

### Шаг 2: Анализ транскрипта

Извлечь:
- **Тип встречи:** 1-1 / комитет / сессия / другое
- **Участники:** имена, роли, компании
- **Решения:** что решили, кто спонсор
- **Задачи:** что, кто, когда
- **Неизвестные термины:** сокращения, внутренние названия
- **Открытые вопросы:** что не решено

### Шаг 3: Интерактивные уточнения

**Когда спрашивать:**
- Неизвестный термин → "Что такое X?"
- Неясный ответственный → "Кто выполнит X?"
- Подтверждение → "Иванов должен X. Верно?"
- Неуверенное имя → "Это Алексей Галактионов?"

**Формат вопросов:**
```
❓ Уточнения:

1. "KMS" — что это?
2. Павел Зыков должен выполнить перенос — верно?
3. Кто ответственный за "подготовку документации"?
```

### Шаг 4: Распределение по правилам

> Использовать rules из meeting-debrief skill

**Автоматически:**
- Создать/обновить профили участников
- Создать файл встречи по шаблону
- Добавить задачи в профили ответственных
- Зафиксировать решения
- Добавить термины в словарь

### Шаг 5: Финальный диалог

```
✅ Готово!

## Что сделано:
- Создана запись встречи: 20_MEETINGS/ktalk/2026-02-27_Sprint-Planning.md
- Обновлён профиль: 10_PEOPLE/pzykov/pzykov.md (2 задачи)
- Создан профиль: 10_PEOPLE/akozlov/akozlov.md
- Добавлен термин: KMS → 50_KNOWLEDGE/glossary.md

---

📝 У вас есть минутки с этой встречи?
[Да, прикреплю файл] [Нет, минуток нет]

💬 Хотите что-то добавить или исправить?
[свободный текст или "всё верно"]
```

---

## Правила распределения

| Сущность | Куда |
|----------|------|
| Новый участник | `10_PEOPLE/{id}/{id}.md` |
| 1-1 встреча | `10_PEOPLE/{id}/1-1/{date}.md` |
| Групповая встреча | `20_MEETINGS/ktalk/{date}_{title}.md` |
| Задача | В файл встречи + профиль ответственного |
| Решение (важное) | `40_DECISIONS/records/DEC-{NNNN}.md` |
| Решение (обычное) | В файл встречи + профиль спонсора |
| Термин | `50_KNOWLEDGE/glossary.md` |

---

## Схема ID

```
ID = {первая_буква_имени}{фамилия_транслит}
```

Примеры:
- Демьянов Максим → `mdemyanov`
- Зыков Павел → `pzykov`
- Козлов Андрей → `akozlov`
