## 2. Концепция проекта

Концепция проекта заключается в том, чтобы создать единый модуль мультиканальности.

Это предполагает работу с модулем из нескольких каналов:
1. Нативное приложение ITSM. По сути просто оболочка для SDK
2. SDK, которое встраивается в приложение клиента.
3. Telegram mini apps. Откроется вьюшка, где под капотом наш SDK
4. MAX — мессенджер. Функциональность пока под вопросом
5. Mattermost. Как — хз, но есть плагин от конкурента https://mattermost.com/marketplace/servicenow-virtual-agent/

**В чём ценность для конечного пользователя:** Он может начать работу в одном канале, а затем продолжить её через любой другой не теряя контекста.

Конкретные преимущества:
1. Продолжить диалог с того места, где остановился.
2. Есть история переписки, а значит и актуальный контекст по заявке
3. Одинаковый пользовательский опыт в каждом из каналов общения

**В чём ценность для компании-клиента:** Сотрудники получают и оказывают услуги в любом удобном канале, который повсеместно внедрён в компании.

Конкретные преимущества:
1. Сотрудники остаются в привычной среде
2. За счёт первого пункта снижается время онбординга в инструмент
3. Возможность кастомизации под брендинг клиента. Канал органично ложится в инфраструктуру. Возможность кастомизации делалась Ильёй в рамках поддержки цветовых тем SMP и SDK 2.0
