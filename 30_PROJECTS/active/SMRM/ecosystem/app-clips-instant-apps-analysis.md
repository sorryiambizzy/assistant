# App Clips и Instant Apps в Enterprise: Анализ применимости

> **Вывод:** Технологии App Clips и Instant Apps **не подходят** для enterprise Service Desk сценариев с многошаговыми процессами и продолжительным доступом.

## Критические проблемы для Enterprise

### 1. Отсутствие идентификации пользователя

| Проблема | Почему это критично |
|----------|-------------------|
| Не знаем кто пользователь | Нельзя создать заявку с корректным автором |
| Нет контекста организации | Нельзя применить настройки роли |
| Нет прав доступа | Нельзя определить доступные услуги |

### 2. Отсутствие continued access

| Ограничение | Значение |
|-------------|----------|
| Время жизни App Clip | Максимум **8 часов** |
| Удаление | Автоматически после использования |
| Доступ к статусу | Теряется вместе с App Clip |

Пользователь создаёт заявку → через 8 часов App Clip исчез → нет доступа к статусу, комментариям,附件.

### 3. Проблема авторизации

```
App Clip без авторизации → Кто пользователь? → Неизвестно
App Clip с SSO → Переключение между приложениями → UX разрушается
```

| Механизм | Проблема |
|----------|----------|
| SFSafariViewController в App Clip | Работает, но App Clip исчезает через 8 часов |
| Переброс в full app | Требует установки |
| Анонимная подача | Нет продолженного доступа к заявке |

### 4. Ограничения конфиденциальности

| Проблема | Почему это проблема |
|----------|---------------------|
| Исполнители заявок | Не видны анонимным пользователям |
| Комментарии по заявке | Содержат внутреннюю информацию |
| История изменений | Конфиденциальна для внешних пользователей |

## Честный вывод по сценариям

### ❌ НЕ работает (Enterprise Service Desk)

| Сценарий | Почему не работает |
|----------|-------------------|
| Быстрая регистрация инцидента сотрудником | Нет идентификации → нет контекста роли |
| Отслеживание статуса заявки | App Clip исчезает через 8 часов |
| Общение с исполнителем | Конфиденциальная информация |
| Многошаговые процессы | Нет continued access |
