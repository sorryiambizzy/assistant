## 2. Контекст и ограничения

**Предпосылки:**
- SMP Mobile — зрелый продукт, разрабатывается командой сервиса мобильных технологий
- Новых пожеланий на развитие от клиентов нет
- Исследование стейкхолдеров показало отсутствие прямых запросов на платные доработки
- Требуется повысить окупаемость продукта
- Решение руководства: довести продукт до уровня поддержки командой сервиса

**Ограничения:**
- Жёсткие сроки: список блоков к 23.03, декомпозиция к 27.03
- Возможный уход команды на проект внедрения в экосистему
- Риск отказа от большого объёма доработок со стороны руководства
- **iOS 26:** необходимо осуществить переезд в течение года для возможности публикации хотфиксов

**Риски:**

| Риск | Вероятность | Влияние | Митигация |
|------|-------------|---------|-----------|
| Невозможность публикации новых сборок iOS из-за просроченного перехода на iOS 26 | Высокая | Критическое | Приоритет №1 для iOS команды |
| Тревожность команды ("нет развития = уволят") | Высокая | Высокое | Раняя коммуникация, прояснение планов на команду (внедрение в экосистему) |
| Список доработок слишком большой (годы реализации) | Средняя | Высокое | Приоритизация по ROI, разделение на фазы, согласование объёма с руководством заранее |
| Скрытые техдолги, не выявленные на этапе планирования | Средняя | Среднее | Технический аудит/code review перед стартом |
| Потеря ключевых экспертов до завершения доработок | Низкая | Высокое | Документирование знаний, перекрестное обучение |
| Изменение приоритетов со стороны бизнеса в процессе | Средняя | Среднее | Жёсткая фиксация scope, изменения только через formal change request |
| Недостаток ресурсов на поддержку после заморозки | Средняя | Среднее | Оценка нагрузки на поддержку заранее, SLA договорённости |

### Ключевые решения от 02.06.2026

1. **Android — на табы целиком:** выпиливаем старую навигацию (боковое меню), не поддерживаем 2 реализации параллельно
2. **Демо-витрина SDK 2 — отдельное приложение** с зашитым приватным ключом; публичный ключ клиент вставляет в настройках своей витрины
3. **Демо-витрина — Deeplinks + QR-коды** (Иван подсмотрел реализацию в Codex)
4. **Кастомизация витрины** (цвета и т.д.) — есть в Chat UI Kit, делаем итерационно после базовой авторизации
5. **Звонилка** при 4-дневке — оценка ~6.5 мес → до конца лета не закроем, основной исполнитель Олег, возможно подключим Яшу
6. **Катю Коновалову** подключаем как аналитика на UX демо-витрины

### Ключевые решения от 01.06.2026

1. **iOS может выпустить табы раньше Android** — допустимое временное расхождение
2. **После SDK 2 Олег → табы Android** (~1.5 мес)
3. **Артём** после стабилизации SDK 2 → фичи + техдолг (websockets/Илья)
4. **Яша** остаётся на звонилке
5. **Идея пилота ЛК для экосистемы** — у iOS проседает нагрузка, через Антона заходить с предложением
