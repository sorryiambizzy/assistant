## Текст письма

**Тема:** RE: Дорожная карта мобильных платформ NCC — сроки, допущения, открытые решения

Коллеги, добрый день!

Прикладываю актуальные сроки по тем активностям, где это возможно

### iOS

1. **SDK 2.0 для iOS — 30.06.2026.** Срок не двигается, выход на Магнит по плану. Техдолг по iOS фактически закрыт (ребята делали его в окнах между задачами), остаётся документация — она вкатывается в регресс и на критический путь не влияет.

2. **Демо-приложение на TestFlight — 30.06.2026.** Приложение готово к пушу, заливаем на этой неделе. Публикация в TestFlight и проверка займет  ориентировочно **3-4 дня**, и этот шаг не в нашей власти (проходит мягкую валидацию Apple). Для валидации поднимаем публичный стенд витрины на инфраструктуре NCC (задача заведена в DevOps NCC).

3. **Киллер-фичи iOS — дату пока не называем.** Скоуп фич определяет А. Гусев, он в отпуске до 16.06 включительно. Сейчас из общего объема известно только **приветственное сообщение**, и его постановка ещё не провалидирована разработкой (возможны доработки на стороне сервера). Дам срок после фиксации скоупа и ревью аналитики — для этого нужен выделенный аналитик. 
Также можем предложить две задачи, по которым обращались клиенты
(1) счётчик непрочитанных — push-подписка вместо pull-метода SDK 1; 
(2) приветственное сообщение — автоотправка от имени пользователя при заходе в чат из раздела приложения (готова к проработке аналитикой).

### Android

1. **SDK 2.0 для Android — 20.07.2026.** Здесь корректирую относительно июня: задачи уходят в тестирование, и реалистичная дата готовности к дистрибуции (регресс + устранение замечаний + документация, рефакторинг/техдолг параллельно) — 20 июля. К этой дате SDK готов к передаче клиенту. 

2. **Звонилка Android — оценка в процессе** (внутри озвученного вами диапазона). Базовая оценка по исследованию Ивана — 40 р.д. разработки + 8 р.д. тестирования; с учётом неопределённости интеграции и новизны WebRTC оценку актуализирую с учетом опыта SDK 2., Срок реалистичен при условиях:
**Ресурсы на консультацию по интеграции SDK ↔ VoIP:** аналитик звонилки  и разработчик API. Также для рбаоты потребуется спецификация от серверных разработчиков. До нее можем начать работы по проработке UI для звонилки.
   - Конкуренция за ресурс с техдолгом Android и с непредсказуемой нагрузкой по дефектам SDK 1.
   - Разработка будет вестись двумя разработчиками, нагрузку распределим между ними

3. **Киллер-фичи Android — дату пока не называем.** Срок зависит от скоупа от А. Гусева. Срок дам после (а) фиксации скоупа, (б) подготовки аналитики и (в) оценки разработки по задачам аналитики.

### Вне плана


### Требуется ваше / А. Гусева решение — поддержка iOS 26 (Liquid Glass) в SDK 1

Пришёл запрос от **ЦБ** на поддержку нового UI Apple (Liquid Glass) в SDK 1. Сейчас задача не в работе — ожидаю от вас решения. Для принятия решения укажу вводные:

- **Трудозатраты:** более 16 р.д. (≈ месяц на человека). Точную оценку готовлю к 16.06. Это прямой вычет из ресурса под SDK 2.0 .
- **Тех.особенности:** в 2027 Apple уберет возможность готовить сборку на старом UI. Без адаптации UI будет выглядеть неаккуратно. Проблем с публикацией приложения и обновлением для клиентов не будет


### Общие допущения, влияющие на все сроки

- Сотрудники работают на неполной занятости — оценки даны под реальную загрузку, не идеальный FTE.
- Действуют задачи по поддержке SDK 1 (дефекты) — могут влиять на сроки SDK 2.0 и звонилки.
- Возможны дополнительные задачи от SMRM — при появлении конкурируют за ресурс мобильной команды. По одному разработчику уйдут на задачи SMP к концу июня. Они не участвуют в стабилизации релиза. Их нагрзука учтена при планировании
- В оценке учтены запланированные сотрудниками отпуска.

### Что прошу с вашей стороны
1. **Скоуп киллер-фич** от А. Гусева по выходу из отпуска (16.06) — после этого датирую киллер-фичи iOS и Android.
2. **Ресурс аналитики** — на проработку киллер-фич и на проработку интеграции звонилки.
3. **Ресурс серверных разработчиков** — на подготовку спецификации и консультации в проекте звонилки
3. **Решение по поддержке iOS 26 в SDK 1** — идём или держим стратегию перехода на SDK 2.
