## 3. Контекст и ограничения

**Предпосылки:**
- AI-технологии достигли уровня для практического применения
- Руководство ставит цели по AI Adoption
- Команды уже начали индивидуальные эксперименты
- Аналитики тратят время на рутинные операции

**Ограничения:**
- Необходимо синхронизировать усилия (избежать дублирования)
- Сквозные процессы должны двигаться централизованно
- Личные ассистенты — в рамках команд
- Необходимо изучить текущие процессы аналитиков

**Риски:**

| Риск | Вероятность | Влияние | Митигация |
|------|-------------|---------|-----------|
| Дублирование усилий | Высокая | Среднее | Регулярные синхронизации |
| Инструмент ради инструмента | Средняя | Среднее | Фокус на боли, а не на технологиях |
| **Отсутствие стандарта Git workflow** | Высокая | Высокое | Воркшоп по Git-модели до переезда |
| **Неясный финансовый статус Gramax** | Средняя | Высокое | Прояснить условия лицензирования |
| **Нет владельца процесса управления требованиями** | Высокая | Высокое | Назначить ответственного |
| ИИ-агент не даст ожидаемой экономии | Средняя | Среднее | Пилотирование на ограниченном наборе |
| Сложность интеграции с процессами | Средняя | Среднее | Постепенное внедрение |
| Проблемы с безопасностью данных | Низкая | Высокое | Аудит безопасности |

**Ответственный за метрики:** Евгений Подкин (сбор, обработка, отчётность)

### Целевые метрики

| Метрика | Базовое значение | Целевой результат | Как измерять |
|---------|-----------------|-------------------|--------------|
| **Среднее время поиска ответа** | TBD (исходное) | -30% | Опрос support, хронометраж |
| **Скорость обновления docs** | TBD | < 1 рабочего дня | Время от issue → обновлена docs |
| **NPS документации** | TBD | 7+ из 10 | Регулярный опрос support/разработчиков |
| **Актуальность информации** | % устаревшей info | < 5% | Аудит документации |
| **Использование (engagement)** | Количество просмотров | +50% | Analytics Gramax |

### График мониторинга

- **Еженедельно** (каждый понедельник): сбор данных по использованию и feedback
- **Ежемесячно** (последний понедельник): отчёт с анализом метрик и рекомендациями
- **Критические события** (сразу): если выявлены проблемы с доступностью или актуальностью

### Риски и точки контроля

| Риск | Сигнал проблемы | Действие |
|------|-----------------|---------|
| Git workflow усложнился | Время на merge > 2 часов | Синхрон с Аней/Денисом по process |
| Поиск неэффективен | NPS < 5 | Анализ, переделка структуры или инструмента |
| Синхронизация dev-docs | Количество устаревших > 10% | Ревью процесса обновления с Таней |
| Граматокс не готов | Частые сбои/недоступность | Обсуждение альтернатив с Максимом |
