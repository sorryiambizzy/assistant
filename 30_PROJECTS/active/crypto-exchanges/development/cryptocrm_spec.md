## 4. Мерчанты и автовыплаты
#### Оценка 80ч

Изначально обменник работает в ручном режиме. Криптообменник самостоятельно проверяет приём средств от клиента и вручную выплачивает средства. Для автоматизации работы обменника можно подключить прием и выплату денежных средств с участием мерчантов — платежных систем, предоставляющих свои кошельки для средств криптообменника. 
Активные мерчанты отображаются в настройках направлений обмена

**Мерчанты (приём средств)**

Мерчант — интрагерация с платженой системой, которая позволяет принимать средства от клиентов на счета криптообменника в платежной системе. Каждый отдельный мерчант - новая интеграция с новой платженой системой. Под каждого мерчанта потребуется отдельная интеграция. Список платежных систем premium exchanger -  https://premium.gitbook.io/main/osnovnye-nastroiki/merchanty-i-avtovyplaty/merchanty


Мерчант может работать в трех режимах: 
- Ручной режим (свои реквизиты)
- Полуавтоматический (приём через мерчант)
- Полностью автоматический (приём и выплата через мерчанты)

Настройки мерчанта:
- Выбор платёжной системы
- API ключи и реквизиты
- Суточный лимит
- Месячный лимит
- Макс. сумма для одной заявки
- Примечание к платежу (что будет передано в комментари к платежну для идентификации транзакции)
- Индивидуальная инструкция к мерчанту, которая отображается на витрине 



**Автовыплаты (выплата средств)**

Автовыплаты работают только в подключенных мерчантах. 

Настройки автовыплаты:
- Выбор платёжной системы
- Автоматическая выплата (да/нет/по умолчанию)
- Суточный лимит
- Месячный лимит
- Мин./Макс. сумма для заявки
- Задержка автовыплаты (в часах)
- Задержка для:
  - Всех пользователей
  - Незарегистрированных
  - Новичков

## 7. Верификация и безопасность (KYC и AML)
#### Оценка 40ч

**7.1 Верификация пользователей**

Верификация личности:
- Загрузка документов на витрине
- Список и типы документов задаются а админке
- Статусы верифкации

Верификация счёта:
- Проверка кошелька в ПС (как работает автоматизация пока не знаю)
- Требование к ручной верификации задается создается вручную путем написания инструкции в админке. Т.е. это выполнение определенных действий пользователем, которые убедят админа в правдивости данных

E-mail верификация:
- Отправка кода на e-mail
- Настройки шаблонов писем

SMS верификация:
- Отправка кода по SMS
- Настройки SMS-шлюза

**7.2 Верификация транзации**

Для проверки используется интеграция с одним из AML-сервисов. Проверка обычно запускается перед созданием заявки 

Интеграция с сервисом:
- ID в сервиса
- Ключ в сервиса 

Что идет на вход:
- Адрес кошелька 
- Адрес транзакции 

Что на выходе:
- Индекс риска (0-100)

На основе значения индекса можно задать номер при котором автоматически будут блокироваться заявки 


**7.3 Ограничения и проверки**

Географические ограничения:
- Запрещённые страны
- Разрешённые страны
- Интергация с геосервисами для определения геопозиции

Лимиты на создания заявок:
- Макс. кол-во заявок с одного IP в сутки
- Макс. кол-во заявок с одного счёта в сутки
- Макс. кол-во заявок от одного пользователя в сутки
- Макс. кол-во заявок с одного e-mail в сутки
- Максимальное количество активных заявок

Другие ограничения:
- Макс. сумма для новичка
- Лимит резерва для направления
- Проверка IP в blacklist

## 14. Витрина (Showcase)
#### Итого по разделу: 128ч (только МВП-скоуп)

Витрина — публичная часть обменного пункта, доступная для неавторизованных пользователей. Основная цель — предоставить информацию о направлениях обмена, курсах и резервах, а также привлечь клиентов к регистрации и созданию заявок.
По сути нам нужно либо организовать cms модуль, либо сделать интеграцию с другими cms.

### Скоуп МВП

В МВП реализуем два модуля. Остальной функционал витрины (главный экран, список направлений, авторизация, статические страницы и т.д.) реализуется средствами внешней CMS / коробочного решения и не входит в разработку — описание ниже сохранено как референс.

### 14.1 Модуль передачи курсов обмена во внешние системы
#### Оценка 8ч

- Выгрузка актуальных курсов обмена во внешние системы (агрегаторы, мониторинги, партнёрские CMS)
- Определение геопозиции пользователя (GeoIP) — для отображения локализованного контента и применения географических ограничений

### 14.2 Модуль подачи заявок неавторизованными пользователями
#### Оценка 120ч

- Форма создания заявки без обязательной регистрации
- Сбор минимально необходимых данных (контакты, реквизиты, направление обмена, сумма)
- Лимиты и проверки для неавторизованных (см. п. 14.9)
- Интеграция с основной воронкой обработки заявок

### Референс: полный функционал витрины (вне МВП)

> Ниже — описание полного функционала витрины. В МВП **не входит**, реализуется через внешнюю CMS или на последующих этапах.

#### 14.3 Главный экран

**Калькулятор курсов обмена:**
- Выбор валюты "Отдаю" из выпадающего списка
- Выбор валюты "Получаю" из выпадающего списка
- Поле ввода суммы для обмена
- Автоматический расчёт суммы получения на основе текущего курса
- Отображение курса обмена для выбранного направления
- Отображение доступного резерва для валюты "Получаю"
- Ссылка на создание заявки (переход к авторизации/регистрации или созданию заявки для авторизованных)

**Фильтрация валют:**
- Категории валют: crypto, stable, rub, usd, eur, CNY, CASH, uah, BYN, KZT, BANK и др.
- Поиск по названию валюты
- Быстрый выбор популярных валют (BTC, ETH, USDT, банки РФ и др.)

#### 14.4 Список направлений обмена

**Табличное представление:**
- Валюта "Отдаю" (с логотипом)
- Валюта "Получаю" (с логотипом)
- Резерв валюты "Получаю"
- Минимальная/максимальная сумма обмена
- Текущий курс обмена
- Статус направления (активно/неактивно)

**Фильтры и сортировка:**
- Фильтр по категориям валют
- Сортировка по резерву, курсу, популярности
- Пагинация или бесконечная прокрутка

#### 14.5 Авторизация и регистрация

**Форма входа:**
- Логин или e-mail
- Пароль
- Персональный PIN-код (опционально)
- Капча (для защиты от ботов)
- Ссылка "Забыли пароль?"

**Форма регистрации:**
- Логин
- E-mail
- Пароль
- Подтверждение пароля
- Согласие с правилами сервиса
- Выбор платёжной системы (по умолчанию)
- Капча

#### 14.6 Информация о сервисе

**Статические страницы:**

- Партнёрская программа

**Информационные блоки:**
- График работы (с 8:00 до 00:00 мск ежедневно)
