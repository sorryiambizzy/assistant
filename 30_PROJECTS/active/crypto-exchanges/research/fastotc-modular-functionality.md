# Модульный функционал FastOTC

> Разбор для команды разработки аналога: функциональные блоки и возможности.

## 1. Модуль «Сделки» (Exchange)

- Создание сделки с выбором направления (что отдаём / что получаем), суммы, валюты, локации.
- Расчёт по курсу (с поддержкой прямого/обратного) или по марже (указывается % или USDT).
- Автоматический пересчёт профита, маржи, суммы справа/слева.
- Закрывающие сделки (контр-сделки) с переключением направления курса.
- Редактирование сделки (суммы, курса, себестоимости) с пересчётом обязательств.
- Автосеттл: привязка тезер-транзакций к сделкам давностью не более 2 дней.
- Ручной матчинг сделок (выбор, с какой сделкой матчить оставшийся объём).
- Фильтрация сделок по комментариям, валютам с обеих сторон, счетам, датам.
- Кнопки внесения/выдачи средств прямо из карточки сделки.

## 2. Модуль «Кассы и балансы» (Locations & Balances)

- Локации (города) с возможностью иметь несколько касс в одной локации.
- В каждой кассе – остатки по разным валютам.
- Переводы между кассами одной локации.
- Балансы контрагентов (клиентов): долг компании – отрицательный, долг клиента – положительный.
- Импорт балансов (через CSV/Excel) с поддержкой черновиков и дельт.
- Выгрузка в Excel (расходы/доходы, переводы, сводка).
- Сортировка контрагентов: сначала с ненулевыми балансами.

## 3. Модуль «Заявки» (Requests)

- Заявки на сделку (предварительный расчёт, без фиксации курса).
- Заявки на пополнение (внесение крипты через инвойс).
- Заявки на выдачу (крипты или фиата, с указанием кошелька/кассы).
- Статусы: ToDo → In Progress → Finished → Settled → Archived.
- Комментарии к заявкам (история с автором).
- Автоматическое создание сделки из заявки (перенос параметров).
- При создании заявки – формирование чата (клиент, кассир, менеджер).

## 4. Модуль «Задачи курьеру» (Courier Tasks)

- Канбан-доска для курьеров/кассиров.
- Статусы с флагом «Деньги получены/переданы».
- Поле «Дата передачи» (дата+время) – заполняется менеджером, курьером или по факту.
- При возврате из Finished в Progress – удаление созданного Settlement.
- Распределение сумм из карточки задачи (без перехода в Settlements).
- Уведомления курьеру/клиенту в Telegram.

## 5. Модуль «Роли и права доступа» (RBAC)

- Предустановленные роли: Владелец, Управляющий, Менеджер, Кассир/Курьер, Клиент.
- Кастомные роли (создание, копирование, изменение прав).
- Права на уровне страниц и действий (CRUD) – можно запретить/разрешить.
- Чувствительные контрагенты – скрытие счетов, сделок, балансов от части пользователей.
- Запрет удаления счёта с дебетовым балансом, запрет дубликатов контрагентов/счетов/валют.
- Шифрование данных брокера (опционально, с отдельным ключом).

## 6. Модуль «Telegram-боты и уведомления»

- Клиентский бот: остатки в USD, последние сделки, создание заявки на сделку/пополнение/вывод.
- Курьерский бот: получение задач, статусы, секретный код, чат с клиентом (прослойка).
- Уведомления о транзакциях, новых заявках, изменении статусов.
- Вход через Telegram (авторизация по диплинку) + 2FA.
- Групповые чаты – бот можно добавить в чат с клиентом и курьером.

## 7. Модуль «Отчёты и дашборды» (Reports)

- Сводка (оптимизированная, с группировкой по дням/кассам).
- Отчёт «Профит» – остатки по валютам с дельтами к предыдущему дню и эквивалентом в USDT.
- Отчёт по операторам – оценка сделок (бонус от -100% до +100%), средняя оценка.
- Отчёт по курьерам – количество выполненных задач, оценка клиентов.
- Дашборд владельца: объём торгов, прибыль, долговая нагрузка, активные клиенты.
- Дашборд управляющего: рентабельность, удержание пользователей, ликвидность.
- Экспорт в Excel (с настраиваемыми колонками).

## 8. Модуль «SAAS / Мастер-база» (Multi-tenant)

- Поддомены для каждого брокера: company.fastotc.org.
- Админ-панель SAAS (мастер-база): список клиентов (брокеров), их статусы, тарифы.
- Управление тарифами (цена, лимит транзакций, набор функций).
- Автоматическое создание базы данных при регистрации нового брокера (через заявку).
- Онбординг: пошаговое заполнение данных (компания → кошельки → кассы → валюты → пользователи → клиенты → остатки).
- Интерактивные подсказки (с запоминанием пройденных шагов).

## 9. Модуль «Платежный шлюз и инвойсы» (Merchant)

- Настройка методов оплаты (криптовалюта + сеть: TRC20, ERC20, BEP20, SOL, TON, ARB, BASE).
- Генерация инвойсов – уникального адреса для каждого клиента (через HD-кошелёк).
- Автоматическое зачисление – слушатель блокчейна отслеживает переводы и зачисляет на баланс клиента.
- Личный счёт брокера – кошелёк, с которого списываются комиссии и расходы на энергию.
- Вывод средств – автоматическая отправка крипты с горячего кошелька по заявке клиента.

## 10. Модуль «Реферальная программа»

- Настройка процентов реферальных отчислений (в CRM, не в админке).
- Многоуровневое дерево (до 5 уровней).
- Анонимизированные ники для клиентов (скрытие реальных имён).
- Отчёт: сколько заработал каждый реферал.

## 11. Модуль «Настройки компании»

- Название, логотип, страна, часовой пояс, контактный email/телефон.
- Выбор валюты отображения (базовая – USDT).
- Настройка источников курсов (Garantex, Grinex, FastOtc.org) для каждой валюты.
- Параметры маржи по умолчанию (по курсам или по марже).
- Таймауты, алерты (курс не обновлялся более 5 минут → тревога).

## 12. Технические сквозные возможности

- Поиск с дебаунсом (отмена предыдущих запросов).
- Пагинация и виртуальная прокрутка для больших списков.
- Оптимизация запросов (кэширование, отложенная загрузка).
- API для внешних вызовов (приёма заявок, получения курсов, статусов).
- Миграции с вербозными сообщениями.
- Логирование всех действий (история изменений сделок, заявок, профита).
