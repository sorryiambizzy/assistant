## Эксперимент с командой Ани (UI 2.0)

> Договорённости с Аней Марковой по запуску Gramax на проекте UI 2.0. Дату синка уточнить.

**Исходные договорённости:**
1. Явной потребности в Gramax на проекте UI 2.0 нет.
2. Есть запрос на **поиск информации из Confluence** — от простого поиска по постановкам до сценария с последующей обработкой найденного.
3. **ЕП** обсудит с Максимом, какую применимость он видит у Gramax и какой эффект ожидает от внедрения.
4. **Аня** запускает внедрение Gramax в проекте UI 2.0 как **эксперимент**.

**Дизайн эксперимента:**
- Тест не на всей документации, а на **одном разделе**. Желательно — с несколькими уровнями вложенности и перелинковкой статей внутри. Раздел выбирает команда Ани.
- Запрос к Gramax — **через Claude Code**.
- Формат тестирования: тест-кейс «вопрос к ИИ → ожидаемый ответ», где ожидаемый ответ — это ответ подготовленного человека в контексте продукта/проекта (эталон).
- Тест-кейсы готовит **команда AI Adoption** на основе выбранного раздела Confluence.
- Ожидаемые ответы готовятся **в синхроне с командой Ани** (убедиться, что эталон — действительно то, к чему должен стремиться ИИ).
- Решение обкатывается на тест-кейсах; «стало лучше/хуже» оценивается по ответам на них.

**Цель тест-кейсов:** проверить, насколько хорошо ИИ-ассистент ищет информацию в Gramax. Требования: нарастают по сложности и адекватны для ИИ — весь контекст вопроса должен лежать в Gramax (в выбранном разделе).

**Статус:** Аня Маркова принесла тест-кейсы (09.06) → ЕП адаптирует и прогоняет. Кейсы получаются сложными — ассистенту не хватает контекста. Раздел Gramax — `https://knowledge.nau.im/ui2/specs` (внутренний ресурс, только из корпсети/VPN).
