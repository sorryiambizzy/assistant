# Индекс: qmd vs собственный MCP-сервер

**Дата:** 04.06.2026
**Статус:** принято

## Контекст

Skill ходит на сайт через curl/WebFetch каждый раз — это медленно, тратит токены и зависит от сети. Нужен локальный индекс. Варианты:

- **qmd** — уже установленный CLI с гибридным BM25 + векторным поиском, MCP-сервером (`qmd mcp`), markdown-friendly. Используется в текущем vault для личного поиска.
- **Собственный MCP-сервер** на Python: SQLite + sqlite-vec (векторы) + FTS5 (BM25) + sentence-transformers.
- **Cloud-сервис**: Algolia DocSearch, Typesense, Meilisearch — отброшено сразу (хостинг публичных копий доки за рубежом, лишние ограничения).

## Решение

Индексируем через **qmd**. Свой MCP — только если qmd упрётся.

## Аргументы

- qmd уже работает у владельца проекта и подключён к Claude Code через MCP.
- Тот же интерфейс поиска, что для личного vault — ниже когнитивная нагрузка.
- Markdown-чанки с frontmatter — единый формат для всего, что индексируется.
- Не нужно писать свой код: краулер + чанкер → стандартный пайплайн → `qmd update && qmd embed`.
- На случай раскатки коллегам qmd ставится одной командой и не требует серверной инфраструктуры.

## Что qmd не делает

- Нет фильтрации по произвольным frontmatter-полям (например, по `source_versions`). На discovery не критично.
- Нет shared-индекса между разными машинами — каждый строит свой. На discovery норм; для команды >5 — узкое место.

## Триггеры пересмотра (когда писать свой MCP)

- Нужна фильтрация по версии SMP / по продукту / по правам доступа.
- Нужен централизованный индекс на сервере, обновляемый автоматически.
- Нужно объединять источники с разной структурой (Confluence + публичная дока + Jira).
- qmd не справится с объёмом (сейчас 49 МБ индекса на 1852 чанка — далеко до проблем).

## Альтернативы, которые рассмотрели

- **sqlite-vec + FTS5 руками** — отброшено, дублирование функций qmd без выгод.
- **LanceDB / ChromaDB** — отброшено, тяжелее в установке для коллег.
- **Локальный Elastic / Meilisearch** — отброшено, серверный процесс с отдельным жизненным циклом.
