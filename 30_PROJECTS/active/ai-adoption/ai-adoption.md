## 3. Контекст и ограничения

**Предпосылки:**
- AI-технологии достигли уровня для практического применения
- Руководство ставит цели по AI Adoption
- Команды уже начали индивидуальные эксперименты
- Аналитики тратят время на рутинные операции

**Ограничения:**
- Необходимо синхронизировать усилия (избежать дублирования)
- Сквозные процессы должны двигаться централизованно
- Личные ассистенты — в рамках команд
- Необходимо изучить текущие процессы аналитиков

**Риски:**

| Риск | Вероятность | Влияние | Митигация |
|------|-------------|---------|-----------|
| Дублирование усилий | Высокая | Среднее | Регулярные синхронизации |
| Инструмент ради инструмента | Средняя | Среднее | Фокус на боли, а не на технологиях |
| Отсутствие стандарта Git workflow | Высокая | Высокое | Воркшоп по Git-модели до переезда |
| Неясный финансовый статус Gramax | Средняя | Высокое | Прояснить условия лицензирования |
| Нет владельца процесса управления требованиями | Высокая | Высокое | Назначить ответственного |
| ИИ-агент не даст ожидаемой экономии | Средняя | Среднее | Пилотирование на ограниченном наборе |
| Сложность интеграции с процессами | Средняя | Среднее | Постепенное внедрение |
| Проблемы с безопасностью данных | Низкая | Высокое | Аудит безопасности |

### Ключевые решения от 02.06.2026

1. **Версия SMP для Gramax-пилота:** 4.21.0 (Костя Бобров обновляет на неё всех клиентов)
2. **ITSM-документация не подходит** (процессная ITSMовская, не SMP) — выгружаем публичную доку SMP
3. **Шаблоны и чек-листы ассистента аналитика:** каждый отдел выделяет инициативного человека (Платформа — Катя Коновалова, Мобильные — Ирина Белова, UI 20 / доп. модули — ЕП напишет)
4. **Шаблоны принимаем без PR-процесса** — скидывают Ирине, она добавляет
5. **Инициативная группа от ТП:** 2-3 человека от Кости Боброва тестируют встроенный поиск Грамокса (агентов у поддержки нет)
6. **Чат AI Adoption:** аватарка — мем Ирины, название — "Уроки AI для аналитиков"
7. **BPM-помощник Антона:** 1-я версия в маркетплейсе, доработан скилл (приоритет код/мета, явный отказ от догадок)

### Ключевые решения от 01.06.2026

1. **Фокус Gramax-пилота:** Тестируем Gramax именно как инструмент поиска по документации (а не процесс вокруг него)
2. **Версия для пилота:** Берём одну из версий SMP — наиболее популярную на поддержке — и сгружаем её в Gramax (по настоянию Максима)
3. **Два канала теста поиска:**
   - Через специалистов ТП — напрямую в Gramax
   - Через ИИ-ассистента (поиск — Антон, подключение к аску — Даша)
4. **Без актуализации до переезда:** Документация не актуализируется до полноценного переезда — чтобы не закручивать масштабный переезд раньше времени
5. **Договорённость с Таней Санниковой:** Исследование направлено на сам инструмент Gramax; процесс вокруг него прорабатываем, только если инструмент себя зарекомендует
